# App Store Release Checklist

**Status:** secondary path, not current release priority
**Updated:** April 8, 2026

## Current Position

The App Store is not the main release strategy for Balls of Steel.

The preferred order is:

1. direct-distributed macOS app
2. private playtesting
3. signing and notarization
4. App Store only if it still makes strategic sense later

Keep this checklist as a reference, not as the main plan.

## If The Mac App Store Ever Becomes Worth It

Make sure all of this is true first:

- [ ] the direct-distribution product already feels complete
- [ ] the app story still makes sense in an App Store environment
- [ ] no core workflow depends on a broker integration the store path would complicate
- [ ] the product can be marketed honestly without pretending to auto-trade
- [ ] the app is narrow enough that it will not get buried under generic finance clutter

## macOS App Store Readiness

### Packaging

- [ ] final app icon exists in the asset catalog
- [ ] bundle identifier is locked
- [ ] version/build numbers are stable
- [ ] code signing is configured correctly
- [ ] app archives cleanly from Xcode

### Compliance

- [ ] privacy policy exists
- [ ] terms/disclaimer language exists
- [ ] product copy does not imply guaranteed performance
- [ ] product copy does not imply live execution where it does not exist
- [ ] local-data handling is described clearly

### Product Presentation

- [ ] screenshots reflect the current SPY and VXX desk workflow
- [ ] screenshots look like the current branded UI, not the old utility version
- [ ] description frames the app as a discretionary trading desk
- [ ] the narrative stays honest about manual-first operation

### Product Fit

- [ ] App Store distribution still looks like an advantage rather than a compromise
- [ ] the audience can actually find the product there
- [ ] the store path does not force the product into a dumber shape

## Current Recommendation

Do not optimize for the App Store until:

- the direct-distributed product is polished
- the private playtest loop is strong
- the data and execution architecture are cleaner
- the release workflow is painless

Right now, the better move is to make the Mac app bundle excellent.
